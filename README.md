
# FastAPI Webhook & File Receiver

هذا تطبيق ويب خلفي (Backend) خفيف جداً مبني باستخدام FastAPI، مصمم لاستقبال البيانات النصية والملفات من أجهزة خارجية أو تطبيقات أخرى.

## المميزات
- **نقطة استلام (Webhook):** يستقبل البيانات عبر `POST /upload`.
- **دعم الملفات:** يستقبل الملفات والصور بصيغة `multipart/form-data`.
- **التوثيق التلقائي:** يقوم بتغيير اسم الملف ليحتوي على التاريخ والوقت (Timestamp).
- **سجلات النظام (Logs):** يطبع البيانات المستلمة في الكونسول لتراها في لوحة تحكم Render.
- **صامت وسريع:** ردود بصيغة JSON فقط.

## المسارات (Endpoints)

### 1. فحص الحالة (Health Check)
- **المسار:** `GET /`
- **الوظيفة:** التأكد من أن السيرفر يعمل.

### 2. رفع البيانات والملفات (Upload)
- **المسار:** `POST /upload`
- **نوع البيانات:** `multipart/form-data`
- **الحقول المطلوبة:**
  - `info`: نص وصفي أو بيانات (String).
  - `file`: الملف المرفوع (File).

## كيفية الاختبار باستخدام cURL
يمكنك اختبار التطبيق من جهازك الشخصي باستخدام الأمر التالي (استبدل `YOUR_URL` برابط Render الخاص بك):

```bash
curl -X POST "https://YOUR_URL.onrender.com/upload" \
     -F "info=Test data from my device" \
     -F "file=@/path/to/your/image.png"
```

## ملاحظة هامة لمستخدمي Render (الخطة المجانية)
- يتم تخزين الملفات في مجلد `uploads/` داخل السيرفر.
- **تنبيه:** في الخطة المجانية لـ Render، يتم حذف الملفات المرفوعة عند إعادة تشغيل السيرفر أو تحديث الكود (Ephemeral Storage). البيانات المطبوعة في السجلات (Logs) ستبقى ظاهرة في لوحة تحكم Render.
